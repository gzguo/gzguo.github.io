2-1-3进程控制

基本概念

定义

进程控制就是对系统中的所有进程实施有效的管理。主要是创建进程、撤销进程、实现进程之间状态转换等的功能

如何实现

用原语实现进程控制

因为进程控制的过程中，要进行进程状态的切换，切换的时候需要更改进程控制块的相关信息，更改需要更改对应的好几条信息，为了防止更改的不对应性。我们使用原语来实现进程控制。

原语的特点就是执行期间不能被中断，只能一气呵成。这种不可被中断的操作就是原子操作。

原语是采用关中断指令和开中段指令实现的。如果执行关中断指令之后有外部中断信号进入，那么外部中断信号会被忽略，这就是原语执行不会被中断的原因

关中断和开中断的权限非常大，所以这两个指令只允许在核心态下执行==>原语运行在核心态

进程控制相关原语

无论是什么原语都无非做三类事情：

  1. 更新PCB中的信息
  2. 将PCB插入合适的队列
  3. 分配/回收资源

创建

创建原语：（创建态）

  1. 申请空白PCB
  2. 为新进程分配所需资源
  3. 初始化PCB
  4. 将PCB插入就绪队列

引起进程创建的事件：

  1. 用户登陆
  2. 作业调度
  3. 提供服务
  4. 应用请求

终止

撤销原语：（终止态）

  1. 从PCB集合中找到终止进程的PCB
  2. 若程序正在运行，立即剥夺CPU，将CPU分配给其他进程
  3. 终止其所有子进程
  4. 将该进程拥有的所有资源归还给父进程或者操作系统
  5. 删除PCB

引起进程终止的事件：

  1. 正常结束
  2. 异常结束
  3. 外界干预

阻塞

阻塞原语：（运行到阻塞）

  1. 找到阻塞的进程对应的PCB
  2. 保护进程运行现场，将PCB状态信息设置为”阻塞态“，暂停进程运行
  3. 将PCB插入到相应进程的等待队列

引起进程阻塞的事件：

  1. 需要等待系统分配某种资源
  2. 需要等待相互配合工作的其他进程完成工作

唤醒

唤醒原语：（阻塞到就绪）

  1. 在事件等待队列中找到对应的PCB
  2. 将PCB从等待队列删除，设置进程为就绪态
  3. 将PCB插入到就绪队列，等待被掉度

引起进程唤醒的事件：

  1. 等待事件的发生（因为什么被阻塞就因为什么被唤醒）

切换

切换原语：（运行态到阻塞赛/就绪态 // 就绪态到运行态）

  1. 将运行环境信息存入PCB
  2. PCB移入相应的队列
  3. 选择另一个进程执行，并更新PCB
  4. 根据PCB恢复进程所需的运行环境

引起进程切换的事件：

  1. 当前的进程时间片到
  2. 有更高优先级的进程到达
  3. 当前进程主动阻塞
  4. 当前进程终止
